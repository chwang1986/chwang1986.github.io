四、部署方式

方式一：整个仓库部署（推荐）

  1. 将整个 trae_workhome 文件夹推送到 GitHub 仓库
  2. 在仓库 Settings > Pages 中，选择 Branch: main，Folder: / (root)
  3. 访问地址：https://你的用户名.github.io/仓库名/

入口建议：当前没有根目录 index.html，建议先访问子目录： - https://你的用户名.github.io/仓库名/mac-mini-guide/mac-mini-guide.html - https://你的用户名.github.io/仓库名/ai-guide/index.html - https://你的用户名.github.io/仓库名/fun-physics/index.html

方式二：分别部署为独立站点

如果想让每个网站有独立域名，可以拆分成三个仓库： - mac-mini-guide 仓库 → 用户名.github.io/mac-mini-guide - ai-guide 仓库 → 用户名.github.io/ai-guide - fun-physics 仓库 → 用户名.github.io/fun-physics

五、当前存在的问题与优化建议

问题 1：缺少总入口页面

现状：根目录没有 index.html，直接访问根路径会 404。

建议：创建一个根目录 index.html 作为三个网站的导航门户：

trae_workhome/
├── index.html          ← 新增：总入口，三个大卡片分别链接到三个网站
├── mac-mini-guide/
├── ai-guide/
├── fun-physics/
└── ...

问题 2：字体文件重复

现状：三个网站各复制了一套字体（共 12 个字体文件，约 2-3MB × 3）。

建议：将字体统一放到根目录 _shared/fonts/，各网站引用 ../../_shared/fonts/ 或 ./_shared/fonts/（如果保持独立仓库则不需要改）。

问题 3：fun-physics 使用外链 CSS，其他网站使用内联

现状： - mac-mini-guide、ai-guide：CSS 内联在 <style> 标签中 - fun-physics：引用了 shared.css

建议：统一风格，但不影响部署。

问题 4：Markdown 文件不会自动渲染

现状：notes/ 和根目录的 .md 文件在 GitHub Pages 上默认不会渲染为网页（会显示原始 Markdown 或下载）。

建议： - 如果希望 Markdown 能在网页中查看，可以安装 Jekyll 主题（GitHub Pages 原生支持 Jekyll） - 或者将重要内容转为 HTML 页面

问题 5：图片体积

现状：hero 图片为 AI 生成，单张约 200-500KB，11 张总计约 3-5MB。

建议： - 对于 GitHub Pages 完全可接受（免费版有 1GB 存储和 100GB/月 流量） - 如需优化，可用 TinyPNG 等工具压缩
